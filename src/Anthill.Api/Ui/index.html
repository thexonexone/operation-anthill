Capability Gates

          
Each gate is off-by-default and fail-closed. Turning one on widens what the colony (or an admin) can do — review before enabling on a network-reachable install.

          

        


        

          
Workspace Boundary

          
The only directory the file/coder ants may read from and propose patches against. Paths outside it are always rejected.

          

        


        

          
Operator Shell

          
Controls the admin-only Shell console (Configuration ? Shell) · a direct terminal into this host. This is remote code execution; keep it off on anything you don't fully trust on the network.

          

          

        


        

          
Autonomous Auto-Apply HIGHEST RISK · PHASE 5

          

            Lets the 24/7 Director apply coder patches without human review — but only ones matching the allowlist below, and only if the workspace still builds + tests green afterward (a red verify auto-rolls-back from backup). Inert until you add at least one path glob. Also requires the File Writing + Patch Application gates above.
          

          

          

            

          

            

            

            

          

          

          

            
Git integration commits verified changes to a standalone branch — never main, and can push it via an SSH deploy key (referenced by path on the host; the key is never stored, shown, or logged). Keep the workspace clone checked out on the branch below; ANTHILL refuses to commit if it is on main.
